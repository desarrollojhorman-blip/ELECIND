## 1. Refactor `tipo_dia` → `tipo_hora` (4 valores)

**Decisión:** un albarán representa una jornada; toda la jornada se factura como **un único** tipo de hora, pero ahora hay 4 categorías distintas en lugar de 2.

| Antes | Ahora |
|---|---|
| Enum `TipoDia` (`laborable`, `festivo`) en cabecera | Enum `TipoHora` (`laboral`, `laboral_noche`, `festivo`, `festivo_noche`) en cabecera |
| Columna `albaranes.tipo_dia` | Columna `albaranes.tipo_hora` |
| 2 etiquetas | 4 etiquetas con helpers `esNoche()` y `esFestivo()` |

**Migración:** `2026_05_15_120000_rename_tipo_dia_to_tipo_hora_albaranes.php`
- Renombra columna `tipo_dia` → `tipo_hora`.
- Expande el enum a 4 valores.
- Mapea valores existentes: `laborable` → `laboral`, `festivo` se mantiene.

**Impacto en código:**
- Modelo `Albaran` cast actualizado (`tipo_hora => TipoHora::class`).
- Form `AlbaranForm` cambia `tipo_dia` por `tipo_hora`.
- Vista móvil `crear.blade.php` y `ver.blade.php` muestran `etiqueta()` del nuevo enum.

## 3. Mejoras UX móvil — alta de albaranes

Pequeños ajustes que reducen scroll y aumentan el contexto visible al operario:

### 3.1 Unidad de medida visible junto a Cantidad

En cada línea de material del formulario móvil (`livewire/mobile/albaranes/crear.blade.php`), el input de cantidad muestra a su derecha un badge gris no editable con la `unidad_medida` (`litro`, `metro`, `kg`, `ud`, …) del material seleccionado. Si aún no hay material elegido, muestra `—`.

```blade
<x-mobile.field label="Cantidad" required>
    <div class="flex items-stretch gap-2">
        <div class="min-w-0 flex-1"><x-ui.input type="number" ... /></div>
        <span class="badge">{{ $matSeleccionado?->unidad_medida ?? '—' }}</span>
    </div>
</x-mobile.field>
```

### 3.2 Stock visible en el select de materiales

Cada `<option>` del select de materiales muestra ahora `Descripción | <stock> <unidad>` (ej. `Cable rojo 2.5mm² | 245 metro`). Permite al operario decidir directamente qué material elegir sin tener que abrir el panel de detalles.

> **Limitación HTML conocida:** el `<option>` nativo no admite estilos ni saltos de línea internos. Se descartó la opción de un dropdown custom con Alpine porque la separación con `|` y la abreviatura sin etiqueta `Stock:` (`Material | 245 metro`) cubre el caso con código mínimo.

## 4. Configuración Empresa — logo principal + logo albarán + zoom + ENIA por defecto

**Esta es la pieza más grande de la jornada.** Reescribe la pantalla de configuración de empresa con un sistema escalable de marca corporativa.

### 4.1 Cascada del logo principal

Se establece esta cascada centralizada en `App\Support\Branding::logoUrl()`:

1. **Logo subido por la empresa** (`empresa.logo_path`).
2. **Logo ENIA por defecto** (`public/images/brand/enia.svg`) — para empresas que aún no han subido el suyo.
3. **Texto fallback** con el nombre/abreviatura de la empresa.

Además, si el `<img>` falla al cargar en el navegador (404, archivo corrupto, etc.) se muestra **"Imagen no disponible"** mediante `@error="logoRoto = true"` con Alpine.

### 4.2 Logo separado para albarán / factura

Nuevo campo `empresa.logo_albaran_path` (nullable). Cascada propia:

1. **Logo de albarán específico** (`empresa.logo_albaran_path`).
2. **Cae al logo principal** (con su propia cascada ENIA → fallback).

Helper: `Branding::logoAlbaranUrl()`.

> **Por qué separado:** el logo del PDF puede tener distinta proporción (más detalle, más horizontal, más grande) que el de la cabecera web/móvil. Si la empresa no quiere usar uno distinto, no rellena este campo y todo sigue funcionando con el principal.

### 4.3 Detección automática de ratio aspect

Al subir un logo, se detecta automáticamente su ratio (`ancho / alto`) y se persiste en BD:

- `empresa.logo_ratio` (float, nullable) — para el principal.
- `empresa.logo_albaran_ratio` (float, nullable) — para el de albarán.

Helper: `Branding::detectarRatio(string $absolutePath): ?float`. Soporta:
- **Raster (PNG/JPG/WebP)**: vía `getimagesize()`.
- **SVG**: parseando `viewBox` o atributos `width`+`height` con regex (sin requerir librería XML extra).

### 4.4 Reglas internas por ratio

Constantes en `Branding`:
- `RATIO_CUADRADO_MIN = 0.85`
- `RATIO_CUADRADO_MAX = 1.15`

| Ratio detectado | Tratamiento |
|---|---|
| **0.85 ≤ ratio ≤ 1.15** | Cuadrado → se muestra en TODOS los contextos (incluido sidebar colapsado 32×32). |
| **> 1.15** | Horizontal → en sidebar colapsado se muestra **abreviatura** (ej. `E`); en cabecera expandida se muestra el logo proporcional. |
| **< 0.85** | Vertical → mismo trato que horizontal (abreviatura en colapsado). |
| `null` | Tratado como cuadrado por defecto (ej. logo ENIA si no se pudo detectar). |

Helpers expuestos: `Branding::logoEsCuadrado()`, `Branding::logoAlbaranEsCuadrado()`, `Branding::ratioEsCuadrado(?float $ratio)`.

### 4.5 Zoom configurable por logo

Dos campos nuevos enteros (default 100):
- `empresa.logo_zoom` — para el principal.
- `empresa.logo_albaran_zoom` — para el de albarán.

Validación: `in:80,90,100,110,120,130` (6 niveles discretos en lugar de slider libre, evita decisiones complicadas).

**Aplicación en runtime:** se inyecta vía `style="max-height: calc(2.25rem * {zoom}/100)"` en el sidebar y en el header móvil. No se reprocesa la imagen — solo se cambia el `max-height` del `<img>`.

### 4.6 Migración

`2026_05_16_100000_add_logo_albaran_and_zoom_to_empresa.php` *(idempotente)* añade:
- `logo_ratio` (float nullable, after `logo_path`)
- `logo_zoom` (smallint default 100)
- `logo_albaran_path` (string 255 nullable)
- `logo_albaran_ratio` (float nullable)
- `logo_albaran_zoom` (smallint default 100)

### 4.7 UI nueva en Configuración Empresa

`resources/views/livewire/empresa/edit.blade.php` se rediseña con dos paneles paralelos (logo principal y logo albarán) cada uno con:

- Caja de previsualización (con cascada visual: si albarán está vacío y principal existe, se muestra el principal con badge "Usa el principal").
- Uploader (PNG/JPG/SVG/WebP, máx 2 MB).
- Botones "Quitar / Cancelar / Descartar".
- **Selector de zoom** (80% – 130%).
- **Hint contextual según ratio detectado** (color-coded):
  - ✅ Cuadrado: "Se mostrará entero en todas las vistas."
  - 🔵 Horizontal: "En el sidebar colapsado verás `E` en su lugar."
  - 🔵 Vertical: "Considera un logo más horizontal para cabeceras."
- **Mini-preview** en vivo (cabecera móvil simulada y cabecera PDF simulada) que actualiza al cambiar el zoom.

Además, una caja de **recomendaciones** explica al usuario los formatos óptimos (cuadrado, ≥400×400, sin márgenes transparentes, SVG preferido).

### 4.8 Fallback visual cuando el `<img>` falla

Tanto en sidebar como en header móvil, si la imagen no carga en el navegador:
- Sidebar expandido / header móvil → texto **"Imagen no disponible"**.
- Sidebar colapsado → abreviatura.

Implementado con `x-data="{ logoRoto: false }"` + `@error="logoRoto = true"` (Alpine).
