## 🎯 Lo que se completó en esta iteración (3)

### 🗂️ CRUD `Proyectos` con creación de Tipo al vuelo
- Pantalla `/proyectos` como ítem principal del sidebar.
- Tabla con 7 columnas: Nombre + fechas, Código, Tipo (badge granate), Cliente, Responsable, Estado, Acciones.
- Filtros plegables independientes y combinables: **Estado · Tipo · Cliente · Responsable**.
- Estados con badges semánticos coloreados (Borrador gris, Activo verde, Cerrado azul, Archivado ámbar, Eliminado rojo).
- Modal de alta/edición en grid 2 columnas con todos los campos del modelo.
- **Sub-modal anidado** para crear Tipo de Proyecto al vuelo: select de Tipo + botón verde `+`. Al guardar, el nuevo tipo queda autoseleccionado en el form de proyecto.
- Validaciones:
  - Nombre + Cliente obligatorios.
  - Código único por cliente (mismo código en otro cliente sí permitido).
  - Fecha fin ≥ fecha inicio.
  - Tipo único al crear al vuelo.
- 13 tests feature pasando (incluido el flujo completo de "crear tipo al vuelo y verlo seleccionado").

**Decisión clave**: la pantalla independiente "Tipos de proyecto" fue eliminada en favor de gestión integrada desde el formulario de Proyecto. Modelo, factory, policy y permisos `tipos_proyecto.*` se mantienen para uso desde el form.

### 📦 CRUD `Materiales` (catálogo)
- Pantalla `/materiales`.
- Tabla con 7 columnas: Código, Nombre + descripción, Grupo (badge), **Stock total + ⚠️ alerta**, Stock mínimo, Estado, Acciones.
- **Stock total** se calcula dinámicamente con `withSum('lotes', 'stock_disponible')`.
- **Alerta ámbar** automática si stock total ≤ stock mínimo configurado.
- Filtros: Estado (Activos/Inactivos/Papelera) + Grupo (dinámico desde BD).
- Modal con: Código (único, opcional), Grupo, Nombre, Descripción, Unidad de medida, Stock mínimo, Notificar stock bajo (toggle), Activo (toggle).
- 10 tests feature.

---

### 📄 CRUD `MaterialLotes` (master-detail con migaja)
- Pantalla `/materiales/{id}/lotes` accesible desde el icono 📦 (caja) de cada material.
- **Migaja**: `← Materiales / [nombre del material]`.
- **Cabecera de resumen** con datos del material padre: nombre, código, grupo, descripción, **stock total y stock mínimo destacados** (con tipografía grande).
- Tabla con 7 columnas: Código lote, Proveedor, Nº pedido, **Stock disp/inicial + barra de progreso**, Entrada, Caducidad, Acciones.
- **Barra de progreso visual** por lote (verde >50% / ámbar 1-50% / rojo 0%).
- **Caducidad coloreada**: ámbar si caduca en ≤30 días, rojo si ya caducó.
- Filtros: Todos / Con stock / Sin stock / Papelera.
- Validaciones:
  - Código de lote único globalmente.
  - Stock disponible ≤ stock inicial.
  - Fecha caducidad ≥ fecha entrada.
- 7 tests feature.

## 🎨 Decisiones de diseño confirmadas (no re-debatir)

1. **Tipos de proyecto = grupos dinámicos** (no categorías fijas). Se gestionan al vuelo desde el form de Proyecto, no requieren pantalla propia.
2. **Proyecto = entidad central** (lleva usuarios, genera albaranes); el tipo es solo agrupador secundario.
3. **Master-detail con dos pantallas y migaja** (no modal anidado con tabla) cuando hay relación 1:N relevante. Aplicado en Materiales → Lotes. Patrón replicable.
4. **Paleta semántica universal** para acciones (verde crear / rojo eliminar / azul editar / ámbar avisos / gris neutro). Granate ELECIND solo para identidad de marca (logo, sidebar activo, cabeceras de tabla, badges de marca).
5. **CSS variables overridables** desde futura Configuración empresa: cambiar logo + colores no requiere recompilar.
6. **Toolbar estándar** en TODAS las tablas: `[+ Nuevo] [≡ Acciones▾] [🔍 Buscar] [✕ Limpiar todo] [⌽ Filtros]`. Patrón cerrado.
7. **Acciones inline por fila como iconos** (no texto): lápiz azul (editar), papelera roja (eliminar), flecha curvada verde (restaurar), iconos contextuales (📦 lotes…). Tooltip al hover.
8. **Modales con par "Cancelar" (gris) + "Guardar" (verde)** abajo a la derecha. Header rosa pálido + ✕.
9. **Color del Tipo de proyecto**: descartado. Era ruido visual sin valor.
10. **Inversión en UI base**: el catálogo de 18 componentes Blade reutilizables NO fue desviación. Es la base que acelera cada CRUD nuevo a 30-60 minutos. No re-debatir.
