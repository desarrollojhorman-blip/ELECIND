  - ⚫ **Gris** → cancelar, cerrar sesión, neutros.

### 🛠 Decisiones de modelado tomadas
- **Tipos de proyecto** son **agrupadores dinámicos** de proyectos, no categorías fijas
  (ejemplo del cliente: "Marzo > Marzo-A, Marzo-B, Marzo-C").
- **Proyecto es la entidad central** (lleva usuarios, genera albaranes, etc.); el tipo
  es solo un agrupador secundario.
- **No habrá pantalla independiente para Tipos**: se gestionarán al vuelo desde el
  formulario de Proyecto (select con opción `+ Crear nuevo tipo`).
- La pantalla de Tipos que se construyó en una iteración previa ha sido eliminada,
  pero el modelo `TiposProyecto`, factory, policy y permisos `tipos_proyecto.*` se
  mantienen para uso desde el form de Proyecto.
- **Color del Tipo**: descartado. Era ruido visual sin valor funcional.
- **Configuración empresa** (logo + colores) será una pantalla configurable; los defaults
  ELECIND están hardcodeados en CSS vars hasta entonces.
