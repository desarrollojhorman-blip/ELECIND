## 2. Empresa — inversión de campos + nuevas columnas

### El problema

`empresa.nombre` se usaba en sidebar, cabeceras PDF y mails (vía `Branding::nombre()`), pero contenía la **razón social larga** ("ELECTRICIDAD INDUSTRIAL E INSTALACIONES, S.L.L."). El nombre corto de marca ("ELECIND") vivía en `nombre_comercial`, que casi no se consumía. Inversión natural: `nombre = marca corta`, `razon_social = legal`.

### Migración

[`2026_05_27_140000_invert_nombre_and_add_movil_web_to_empresa.php`](../../database/migrations/2026_05_27_140000_invert_nombre_and_add_movil_web_to_empresa.php):

```sql
ALTER TABLE empresa CHANGE COLUMN nombre razon_social VARCHAR(150) NULL;
ALTER TABLE empresa CHANGE COLUMN nombre_comercial nombre VARCHAR(150) NULL;
ALTER TABLE empresa ADD COLUMN movil VARCHAR(30) NULL AFTER telefono;
ALTER TABLE empresa ADD COLUMN web   VARCHAR(255) NULL AFTER movil;
```

`CHANGE COLUMN` preserva los datos al renombrar — no se perdió ni un valor.

## 4. Correo de solicitud de firma — espejo del PDF (con tablas)

### Cambios visuales

Mismo esqueleto que el PDF: cabecera con empresa completa, datos del cliente en rectángulo, tabla de horas con bordes y divisores, botón "Firmar ahora" en `#1f2937`. **Sin materiales** (decisión del usuario: en el correo y en la pantalla de firma no van; solo en el PDF).

### Cambios técnicos

- **100 % tablas anidadas** (`<table role="presentation">`). La versión inicial usaba `<div>` para el bloque cliente y el botón, lo que rompía en Gmail (collapsing de contenido). El layout basado en tablas es el patrón "bulletproof" estándar para HTML emails.
- **Preheader único por envío** al principio del `<body>`:
  ```html
  <div style="display:none;…">
      Solicitud de firma del albarán {nº} … Ref. {token} · Enviado {timestamp}
      {caracteres invisibles ZWJ para forzar ~150 chars}
  </div>
  ```
  Esto resuelve el problema de Gmail al colapsar correos que considera "auto-respuestas" cuando te los mandas a ti mismo varias veces. Cada envío tiene token y timestamp distintos → Gmail no puede deduplicar.
- Atributos `align="left|center"` directos en `<th>`/`<td>` (más compatibles con Outlook viejo que `text-align`).
- `border="0"` explícito en tablas e imágenes.

## 6. Control de Horas — paginación

### Patrón

Mismo que el resto de listados (Clientes / Conceptos / Albaranes / Borradores / Materiales / Roles):

- `use WithPagination;`
- `#[Url(as: 'pp')] public int $porPagina = 25;`
- Selector "Filas:" con valores `[5, 10, 25, 50, 100, 250, 500]`
- `paginate($porPagina)->onEachSide(2)` con `{{ $lineas->links() }}` arriba de la tabla (no abajo, por petición del usuario).
- URL guarda estado completo (`?pp=50&page=3&trabajador=7&desde=…`).

### Refactor del query

El componente tenía un único `Computed::lineasDiarias()` que hacía `->get()` y devolvía una Collection. Ahora:

- `baseQuery(): QueryBuilder` privado — los joins y `when()` de filtros vive aquí.
- `render()`:
  - **Lista visible** → `baseQuery()->select(…)->paginate($porPagina)`.
  - **Totales** → `baseQuery()->groupBy('tipo_hora')->select(['tipo_hora', SUM(horas), SUM(horas_extra)])->get()`. Una sola query agregada en lugar de sumar en PHP la Collection. Los totales siempre reflejan **todo lo filtrado**, no solo la página visible.

### Reset de página

Hook genérico `updated($name)` que llama a `resetPage()` cuando cambia cualquier filtro o `porPagina`, y mantiene la cascada cliente → proyecto.

### Fórmulas por defecto

- **Laboral / Lab. Noche** → `Importe normales = horas × tasa_hora`
- **Festivo / Fest. Noche** → `Importe normales = horas × tasa_festivo`
- Siempre → `Importe extras = horas_extra × tasa_extra`

Si la fórmula real (por ejemplo, las extras del festivo van con `tasa_festivo`, o hay multiplicadores) es distinta, se ajusta en `HorasExport::map()` sin tocar nada más.

### Filtros y nombre del fichero

El botón **Exportar Excel** del blade pasa los filtros activos como query params (omitiendo vacíos):

```php
route('horas.exportar.excel', array_filter([...], fn ($v) => $v !== null && $v !== ''))
```

Y el controller genera un sufijo dinámico en el nombre del fichero:

- Con rango: `control_horas_2026-05-01_a_2026-05-31.xlsx`
- Solo desde / solo hasta: `control_horas_desde_…` / `control_horas_hasta_…`
- Sin fecha: `control_horas_2026-05-27.xlsx`

### Memorias creadas en esta sesión

- `smtp-gmail-config.md` — host/puerto/cifrado de Gmail SMTP para Ajustes → Correo. **La contraseña NO se guarda en memoria**, solo en BD vía form o en `.env`.

Cuando el envío es a uno mismo o llega varias veces el mismo body, Gmail puede colapsar el contenido. El preheader único con `$tokenFirma->token` + `now()` resuelve la mayoría de los casos. Si vuelve a pasar, primero borrar el hilo anterior para que Gmail no encuentre duplicados.
