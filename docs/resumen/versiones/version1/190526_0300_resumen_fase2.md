## 2. Módulo Borradores — Fase 1 (base completa)

### Decisión de arquitectura (aprobada)
- Tabla separada `borradores` (no se reutiliza tabla albaranes).
- Patrón "columna FK + columna texto libre" para proyecto, cliente, concepto y trabajadores/materiales.
- Vinculo borrador → albarán: campo convertido_a_albaran_id con SET NULL al borrar el albarán.
  Permite reutilizar el borrador para generar un nuevo albarán.
- Transición ONE-WAY: borrador → albarán. No se puede revertir.
- Navegación: submenú bajo "Albaranes" en el sidebar (Borradores | Albaranes).

### Base de datos
Migración: 2026_05_19_100000_create_borradores_tables.php

Tabla borradores:
  id, numero_borrador (único), proyecto_id (nullable FK), proyecto_texto (nullable),
  cliente_id (nullable FK), cliente_texto (nullable),
  concepto_id (nullable FK), concepto_texto (nullable),
  responsable_id (nullable FK), fecha, tipo_hora (enum), estado (pendiente|convertido),
  observaciones, convertido_a_albaran_id (nullable FK SET NULL), creado_por, timestamps, softDeletes

Tabla borrador_lineas_personal:
  id, borrador_id (FK cascade), trabajador_id (nullable FK), trabajador_texto (nullable),
  horas decimal(5,2), horas_extra decimal(5,2), timestamps

Tabla borrador_lineas_material:
  id, borrador_id (FK cascade), material_id (nullable FK), material_texto (nullable),
  cantidad decimal(10,2), timestamps

### NumeracionService
Añadido:
- PLANTILLA_DEFECTO_BORRADOR = 'BOR-{NNNN}'
- plantillaBorrador(): lee empresa.plantilla_numeracion_borrador o usa el default.
- siguienteNumeroBorrador(): secuencial global (sin reinicio anual), dentro de transacción DB.

### Modelos
- app/Models/Borrador.php — tabla borradores, relaciones completas, métodos helper:
    proyectoNombre(), clienteNombre(), conceptoNombre(), estaConvertido()
- app/Models/BorradorLineaPersonal.php — método trabajadorNombre()
- app/Models/BorradorLineaMaterial.php — método materialNombre()

Importante: $table = 'borradores' declarado explícitamente (Laravel pluralizaría a 'borradors').

### Permisos y Policy
Permisos añadidos al seeder y ejecutados:
  borradores.ver_todos, borradores.ver_propios, borradores.crear,
  borradores.modificar, borradores.convertir

app/Policies/BorradorPolicy.php: view, create, update, delete, convertir, restore.
Registrada en AppServiceProvider.

### BorradorForm (Livewire Form Object)
app/Livewire/Forms/BorradorForm.php:
- Propiedades: proyecto_id/texto, cliente_id/texto, concepto_id/texto, responsable_id,
  fecha, tipo_hora, observaciones, lineasPersonal[], lineasMaterial[].
- Validación flexible: solo fecha y tipo_hora son required.
- fromModel(Borrador): carga todas las líneas.
- save(): genera numero_borrador en creación, limpia texto si hay FK, sincroniza líneas (delete+recreate).
- addLineaPersonal/removeLineaPersonal, addLineaMaterial/removeLineaMaterial.

### Componentes Livewire
app/Livewire/Borradores/Index.php:
  Listado con paginación, búsqueda, filtros (estado, desde, hasta), ordenación.
  Búsqueda en numero_borrador, proyecto_texto, cliente_texto, y relaciones FK.
  Acciones: ver, editar, eliminar, restaurar.

app/Livewire/Borradores/Editar.php:
  Crear y editar. Tabs: Borrador | Trabajadores | Materiales.
  Computeds: proyectosDisponibles, clientesDisponibles, conceptosDisponibles,
  trabajadoresDisponibles, materialesDisponibles (sin filtro por proyecto, al ser borrador libre).

app/Livewire/Borradores/Ver.php:
  Vista readonly con tabs: Borrador | Trabajadores | Materiales.
  Alerta si ya está convertido con enlace al albarán generado.
  Acción convertirAAlbaran():
    - Valida proyecto_id y cliente_id sean FK reales (no texto libre).
    - Crea Albaran con todos los datos del borrador.
    - Copia líneas con FK resuelta (ignora texto libre).
    - Marca borrador como convertido y guarda el id del albarán.
    - Redirige al editar del albarán creado.

### Vistas Blade
resources/views/livewire/borradores/
  - index.blade.php: tabla con badge de estado, búsqueda, filtros, modal eliminar.
  - editar.blade.php: campos con toggle "Seleccionar existente | Texto libre" para proyecto,
    cliente, concepto, trabajadores y materiales. Líneas editables inline (no modales).
  - ver.blade.php: readonly, modal eliminar, modal confirmar conversión a albarán.

### Rutas (routes/web.php)
  GET /borradores               → borradores.index    (can:borradores.ver_todos)
  GET /borradores/crear         → borradores.crear    (can:borradores.crear)
  GET /borradores/{borrador}    → borradores.ver      (can:borradores.ver_todos)
  GET /borradores/{id}/editar   → borradores.editar   (can:borradores.ver_todos)

### Sidebar
El item "Albaranes" (antes enlace directo) se convierte en submenú:
  ▸ Albaranes
      ├── Borradores  → borradores.index
      └── Albaranes   → albaranes.index
