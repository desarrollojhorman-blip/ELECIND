## 3. Módulo de Materiales — control de acceso completo

### Qué se hizo
- El sidebar ya ocultaba el bloque de materiales cuando `modulo_materiales_avanzado = false`.
- Se añadió protección a nivel de ruta: todas las rutas `/materiales/*` están dentro de un grupo con middleware `modulo.materiales`.
- El middleware hace `abort(404)` si el módulo está desactivado, así no son accesibles aunque se escriba la URL a mano.

### Archivos
- `app/Http/Middleware/EnsureModuloMateriales.php` (nuevo)
- `bootstrap/app.php` — alias `modulo.materiales` registrado

## 5. Logs de auditoría — `Configuracion\Logs`

### Qué se hizo
- Activado `spatie/laravel-activitylog` (ya estaba instalado, tabla ya existía).
- **8 modelos** loguean automáticamente sus creates/updates/deletes:
  - `Albaran`, `Borrador`, `Cliente`, `Proyecto`, `User`, `Material`, `Empresa`, `ApiToken`.
  - Cada modelo loguea solo los campos relevantes (sin snapshots, sin password, sin token).
  - Solo registra cuando algo realmente cambia (`logOnlyDirty` + `dontSubmitEmptyLogs`).
- **Login y logout** se registran en `LoginController` con la IP del usuario.
- **Visor en `/configuracion/logs`** (permiso `configuracion.ver`):
  - Filtros: búsqueda libre · usuario · tipo de entidad · tipo de evento · rango de fechas.
  - Tabla paginada (50 por página): fecha/hora · usuario · badge de evento · entidad · descripción.
  - Para ediciones muestra el diff campo a campo: `campo: "antes" → "después"`.
  - Solo lectura, sin botones de borrar.
